## Context

当前思源笔记分享插件仅支持官方默认存储后端，用户无法自主选择存储位置。随着用户对数据自主可控需求的增加，需要支持多种自定义存储后端。

**约束条件**：
- 后端服务由另一个项目维护，插件通过 HTTP API 与后端通信
- 敏感信息（Token、SecretKey）需要加密存储
- 需要保持向后兼容，不启用自定义存储时行为不变
- 单文件大小限制 50MB，控制成本和性能

**利益相关者**：
- 终端用户：需要简单易用的存储配置界面
- 开发者：需要清晰的 API 和类型定义

## Goals / Non-Goals

**Goals**：
1. 支持 OpenList 和 RustFS 两种存储后端
2. 提供直观的存储配置和管理界面
3. 支持存储后端之间的平滑切换
4. 支持存储迁移功能（从源存储迁移到目标存储）
5. 敏感信息加密存储，确保安全

**Non-Goals**：
1. 不支持百度网盘（官方 API 不明确）
2. 不实现后端存储适配器（在另一个项目中实现）
3. 不实现分片上传（后续迭代考虑）
4. 不实现实时同步功能

## Decisions

### Decision 1: 存储配置存储在 ShareProConfig 中
- **What**: 将 `storageConfig` 作为可选字段添加到 `ShareProConfig`
- **Why**: 利用现有的配置管理系统，无需创建新的存储机制
- **Alternatives considered**: 单独存储文件 - 增加了复杂性，没有明显收益

### Decision 2: 前端不直接存储敏感信息明文
- **What**: Token 和 SecretKey 等敏感字段通过后端 API 加密存储
- **Why**: 避免敏感信息泄露，符合安全最佳实践
- **Alternatives considered**: 前端加密 - 密钥管理困难，安全性不如后端加密

### Decision 3: 使用联合类型定义存储后端配置
- **What**: `StorageBackendConfig = OpenListConfig | RustFSConfig | DefaultConfig`
- **Why**: 利用 TypeScript 类型系统确保类型安全，编译时检查
- **Alternatives considered**: 通用对象加类型字段 - 失去了类型安全优势

### Decision 4: 健康检查由前端按需触发
- **What**: 不提供自动定时健康检查，由用户手动触发或切换时检查
- **Why**: 减少不必要的网络请求，避免后端压力
- **Alternatives considered**: 自动定时检查 - 增加了复杂性和资源消耗

### Decision 5: 迁移功能仅提供 UI 控制，实际逻辑在后端
- **What**: 前端只负责展示迁移进度和控制操作，实际迁移由后端执行
- **Why**: 迁移可能耗时较长，后端更适合处理长时间运行的任务
- **Alternatives considered**: 前端轮询实现迁移 - 不可靠，前端关闭会中断

## Risks / Trade-offs

| 风险项 | 影响 | 缓解措施 |
|--------|------|----------|
| OpenList 版本兼容性问题 | 中 | 文档标注支持的版本范围，API 调用保持向后兼容 |
| 大文件上传超时 | 中 | 50MB 限制，后续考虑分片上传 |
| 存储空间不足导致迁移失败 | 中 | 迁移前检查目标存储空间，提供清晰的错误提示 |
| 敏感信息泄露 | 高 | 后端加密存储，前端不保留明文，日志中脱敏 |
| 存储切换时上传中断 | 中 | 切换后立即生效，新上传使用新存储，已有资源保持不变 |

## Migration Plan

**配置迁移**：
- 新增字段为可选，现有用户无感知
- 首次打开存储设置时显示引导提示

**数据迁移**：
- 通过专门的迁移功能实现
- 支持暂停、取消、回滚
- 迁移期间不影响正常分享功能

**回滚策略**：
- 迁移前创建文件索引快照
- 支持增量回滚和完整回滚
- 回滚操作在服务端执行
