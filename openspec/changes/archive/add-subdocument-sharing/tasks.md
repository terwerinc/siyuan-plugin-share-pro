## 实施建议

### 优先级

中优先级

### 原因

1. 子文档分享是常见需求，用户经常需要分享整个文档树
2. 实现相对简单，可以使用思源 API 递归获取子文档
3. 与增量分享功能相辅相成，可以提升用户体验

### 建议行动

- 在增量分享功能完成后，优先实现此功能
- 可以与增量分享功能结合，实现更强大的批量分享能力

## 验收标准

1. 子文档分享功能默认启用
2. 用户可以在全局设置中配置是否默认分享子文档
3. 用户可以在单次分享时单独控制是否分享子文档
4. 启用子文档分享时，系统应自动分享该文档下的所有层级子文档
5. 分享后的子文档应能够通过分享链接正常访问
6. 功能应在新UI和旧UI中都能正常使用
7. 对于拥有大量子文档的情况，分享操作应具有良好的性能
8. 子文档树可视化预览工作正常（完全封装在组件内）
9. 支持手动排除特定子文档
10. **移除数量限制配置**，采用智能性能警告（>50个子文档时提示）
11. 扁平化处理功能正常（获取所有子文档，无论深度层级）
12. 分享前预估准确（总数、时间、存储空间）
13. 分享进度实时显示，支持取消操作
14. **超过50个子文档时显示温和性能提示（非强制确认）**
