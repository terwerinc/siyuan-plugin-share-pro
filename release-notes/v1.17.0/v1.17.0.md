# share-pro v1.17.0: Easier Settings, More Reliable Sharing

In `1.16.0`, we focused on sharing completeness: referenced documents, subdocuments, and incremental sharing were all built to keep knowledge from breaking apart during delivery.

With `1.17.0`, we turned our attention back to another equally important question: as features grow, how do we make configuration clearer, upgrades safer, and daily operations easier?

So this release is still about careful refinement. Instead of adding more buttons for the sake of it, we want the existing power to feel easier to find, understand, and trust.

## 🧭 Preferences Now Feel More Like a Real Workspace

Settings used to be spread across multiple panels. As the plugin grew, the harder part was sometimes not configuration itself, but knowing what could be configured and where to find it.

This release reorganizes the experience into one unified preferences workspace with four sections: `General`, `Appearance`, `Sharing`, and `Advanced`.

- General: license details, registration code, and the new share window entry
- Appearance: themes, custom CSS, title cleanup, domain and path settings, plus SEO-related options
- Sharing: subdocuments, referenced documents, AI assistant, info bar, document tree, outline, and access protection
- Advanced: incremental sharing and blacklist management

And the value is not only visual.

Save state, unsaved-change reminders, sync results, and account information are all presented more clearly. The goal is simple: when you open settings, you should feel oriented instead of lost.

## 🤝 Viewer-Side Improvements, Properly Carried by the Publisher Side

On `March 26, 2026`, we published a dedicated article for the viewer-side AI assistant and info bar:

[AI Assistant Launch: Your Knowledge Delivery Partner](https://siyuan.wiki/x/20260326014537-u1yreq6)

In this release, the publishing side now formally carries the matching compatibility switches without making them feel heavy-handed.

You can now control at the global level:

- whether the AI assistant is enabled
- whether the info bar is shown

There is another detail we care deeply about here:
historical data keeps its legacy-compatible default behavior after upgrade, so existing shared pages are not suddenly changed by newly introduced fields. At the same time, newly published data now includes the compatibility information needed to keep the publisher side and the viewer side speaking the same language. Document-level AI assistant settings are also applied more reliably in new share payloads.

Put simply:
new capabilities arrive, but old links stay calm; new options appear, but existing users do not need to relearn the whole system.

## 🔄 Incremental Sharing Feels Smoother

The more often a feature is used, the more small frictions matter.

In `1.17.0`, we kept polishing the incremental sharing flow:

- selection state is reset correctly after refreshing the list
- checked items no longer linger after batch sharing completes
- the “Select All” wording is now properly localized for a more consistent UI

These are not the loudest changes in the release, but if incremental sharing is part of your regular workflow for reports, note updates, or ongoing knowledge maintenance, you will likely feel the difference quickly.

## 💝 Quiet Work That Still Matters

Alongside the visible improvements, we also did some quieter but important work under the hood:

- tightened the data contract between the publisher side and the viewer side, with tests added to reduce upgrade uncertainty
- refactored the settings-related structure to leave more room for future expansion
- prepared design and development documentation for future custom storage capabilities

These changes do not show up as a button on the screen, but they do support the kind of confidence that matters when you keep upgrading over time.

## 🌈 How to Start

Update the plugin to `v1.17.0`.

If you want to feel the most visible parts of this release first, a simple path is:

1. Open Preferences and explore the new workspace layout
2. Check the AI assistant and info bar switches in the “Sharing” section
3. If you use incremental sharing often, try one full round of refresh, filtering, and batch sharing

## 💌 Final Words

If `1.16.0` answered “can sharing be more complete?”, then `1.17.0` is really asking: “can sharing feel more reassuring?”

Clearer settings, safer compatibility, and smoother operations.
These updates are not flashy, but they make a real difference in day-to-day use.

We believe a good sharing tool is not just about sending a document out.
It should also help you feel confident before you send it.

Thank you, as always, for staying with us.
We will keep refining, listening, and taking even the smallest improvements seriously.
